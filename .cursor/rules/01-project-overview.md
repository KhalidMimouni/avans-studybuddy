# StudyBuddy Project Overview

## Goal
StudyBuddy is a full-stack single page web application for students who want to find, create, and manage study opportunities together.

The application allows users to:
- register and log in
- browse courses
- create study groups for courses
- plan study sessions inside study groups
- enroll in or leave study sessions
- view an About page with the case description, ERD, and functional requirements

## Target users
- students who want to join a study group
- students who want to create and manage their own study group
- students who want a clear overview of courses, study groups, and sessions

## Core idea
A user creates an account, browses available courses, creates or joins study groups, and then participates in concrete study sessions. The app should feel practical, simple, and realistic.

## Project constraints
This project must satisfy the school assignment requirements:
- Angular SPA frontend
- NestJS backend
- Nx monorepo setup
- SQL database
- authentication and authorization
- CRUD on the domain
- overview/detail structure
- filtering across the main entities
- About page containing the case description, ERD, and functional requirements
- realistic seed data

## Scope
The first implementation should focus on the MVP:
- authentication
- course overview and detail
- study group CRUD
- study session CRUD
- enrollment actions
- filtering and searching
- About page
